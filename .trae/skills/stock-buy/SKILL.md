# Stock Buy Skill（买入纪律技能）

> 全局红线（仓位上限、止损公式、事件分级）见 [stock-analyst/AGENT.md](../../agents/stock-analyst/AGENT.md)。
> **本 Skill 是 Example 1 铁律的权威源**：任何其他 Skill 引用 `@buy:Example1` 时，以本文件 §四 为准。

## 一、何时调用

- 用户已经完成或参考了 `stock-analysis`，准备**开仓 / 加仓 / 补仓**
- 用户询问"现在能买吗？""买点在哪？""能不能抄底？"
- 用户报告标的下跌一定幅度，希望"摊低成本"（高危场景，必须套用 §四）
- 用户要求计算具体买入价位、止损位与仓位

## 二、5 重买点确认（缺一不可）

| # | 维度 | 量化条件 |
|---|------|----------|
| 1 | 大盘 | 上升或震荡向上趋势，未跌破 60 日均线 |
| 2 | 板块 | 强于大盘（板块/大盘比值近 20 日上行），处"启动 / 主升"阶段 |
| 3 | 个股 K 线 | 站稳 20 / 60 日均线，形态健康（突破 / 回踩不破 / 多头排列） |
| 4 | 量价 | **放量突破** 或 **缩量回踩支撑反弹**，无量价背离 |
| 5 | 风险预算 | 盈亏比 ≥ 2:1，单笔最大亏损 ≤ 总资金 2% |

> 5 条全部满足 → 进入金字塔首仓；缺任一项 → 继续观察，不强行入场。

## 三、金字塔加仓量化触发条件

> 严格遵循 AGENT §3.3 仓位约束：单票 ≤ 20%、单板块 ≤ 40%、现金 ≥ 10%、**只在盈利时加仓**。

| 阶段 | 仓位 | 量化触发条件 | 同步动作 |
|------|------|--------------|----------|
| **首仓 1/3** | 计划仓位 × 1/3 | 5 重确认全部满足 | 设置初始止损（关键支撑下方 3%–5%） |
| **加仓 2/3** | 再 +1/3 | 浮盈 ≥ 5% **且** 突破前期阻力（前高 / 颈线 / 平台箱体上沿）**且** 突破日成交量 ≥ 5 日均量 1.2 倍 | 上移止损至首仓成本附近 |
| **加仓 3/3** | 再 +1/3 | 浮盈 ≥ 10% **且** 站稳新平台 ≥ 3 个交易日 **且** 板块仍处主升 | 上移止损至 20 日均线下方 |

> 任一阶段触发 §四（@buy:Example1）→ **立即冻结全部加仓额度**。

## 四、铁律 @buy:Example1：放量下跌过程中，不要抄底，禁止补仓买入

**规则**：放量下跌过程中，不要抄底，禁止补仓买入。

**解读**：
- "放量下跌"的标准量价语言含义是**主力出货 / 恐慌性抛售**，是趋势转弱或继续下行的强信号
- 此阶段任何抄底/补仓行为均违反"不补亏损、不左侧接刀"的纪律
- 必须等待**缩量企稳 + 右侧确认信号**后再行动

### 4.1 判定条件（满足任一即触发）

1. 当日成交量 ≥ 近 5 日均量的 **1.5 倍**，且收盘较前一日下跌 ≥ **3%**
2. 连续 2 日放量阴线，跌破关键支撑位（20 日均线 / 前低 / 颈线）
3. 个股出现"放量跌停"或"长上影 + 放量下跌"组合形态

### 4.2 正确处置

- **已有持仓**：触发 AGENT §3.4 三类止损（先达到者执行）
- **空仓观察**：保持空仓，移入"观察池"而非"建仓池"
- **资金管理**：禁止动用任何加仓额度（包括金字塔的剩余 1/3、2/3）

### 4.3 错误处置（明令禁止）

- ❌ "跌得多了便宜了，先买点试试" — 违反"不预测、只应对"
- ❌ "摊低成本" — 违反"只在盈利时加仓，不补亏损"
- ❌ "基本面没变化，跌出黄金坑" — 量价为王，价格已用脚投票；基本面在放量下跌中**永远滞后于市场信号**
- ❌ 用"日内 T+0 抢反弹"作为变相加仓 — 同样违反纪律

### 4.4 失效解除条件（同时满足才允许重新评估买点）

1. 量能萎缩至近 5 日均量的 **70%** 以下（缩量企稳）
2. 出现 ≥ 1 根**放量长下影 / 十字星 / 底分型**
3. 大盘与板块同步企稳，未再创新低
4. 站回关键均线（如 20 日均线）至少 2 个交易日

### 4.5 多空对立思考

- 反方逻辑："放量下跌也可能是最后一跌，地量后必有地价"——成立的前提是**衰竭性放量 + 次日缩量收阳**双重确认，而**不是**在放量下跌当日抄底
- 即使最终是"最后一跌"，等右侧确认再介入只少赚 5%–10%，但避免被腰斩，**期望值始终更优**

## 五、买入决策树

```
是否在 AGENT 黑名单 / S 级事件中？
├─ 是 → ❌ 禁止买入
└─ 否 → 是否触发 @buy:Example1？
        ├─ 是 → ❌ 冻结所有买入/加仓
        └─ 否 → 是否首次建仓？
                ├─ 是 → 5 重确认通过？
                │       ├─ 通过 → ✅ 首仓 1/3 + 设止损
                │       └─ 不通过 → ⏸ 继续观察
                └─ 否 → 当前是否盈利？
                        ├─ 否（亏损）→ ❌ 严禁补仓
                        └─ 是（盈利）→ 加仓量化条件（§三）满足？
                                ├─ 是 → ✅ 加仓 1/3 + 上移止损
                                └─ 否 → ⏸ 持有等待
```

## 七、与其他 Skill 的衔接

- 上游：`stock-analysis` 提供研判结论与行业模板 / `stock-selection` 提供候选池
- 下游：买入后 →
  - 出现"放量下跌 / 跌破止损 / 趋势走坏" → 立即移交 `stock-sell`
  - 持仓档案更新 → 写回 AGENT §六

## 八、禁止行为

- ❌ 重述 AGENT 已有红线（仓位上限、止损公式、2% 法则等只引用，不抄写）
- ❌ 给亏损中的标的"摊低成本"建议
- ❌ 在 5 重确认未全部满足时强行给出买入价
- ❌ 用"短线博弈"绕过 §四 铁律
