# Stock Selection Skill（选股技能）

> 全局红线（仓位、风控、事件分级、黑名单）见 [stock-analyst.md](../../agents/stock-analyst.md)。
> 本 Skill 仅产出**候选池**与**入选/淘汰理由**，不出具具体买点（属 stock-buy）。

## 一、何时调用

- 用户没有给出具体标的，希望"挑几只能买的"
- 用户希望按**风格**（红利、成长、龙头）或**主题**（AI、出海、半导体）筛股
- 用户要求**事件型选股**（围绕政策、财报季、资金主线）
- 用户要求**复盘候选池**（前次推荐的标的现在表现如何）
- 在 `stock-analysis` 之前作为前置步骤使用

## 二、核心理念（与 AGENT 红线一致）

- **自顶向下**：大盘趋势走坏时，宁可不选股
- **强者恒强**：只在强势板块中找强势个股，**不在弱势板块抄底**
- **黑名单优先**：财务造假 / 商誉爆雷 / ST / 连续违规减持 → **直接剔除**，不参与排序
- **候选池而非推荐**：本 Skill 输出 3–10 只候选标的，最终是否买入交由 `stock-analysis` + `stock-buy` 二次确认

## 三、三层筛选漏斗

### Layer 1：大盘判断（决定是否选股）

| 大盘状态 | 选股建议 |
|----------|----------|
| 上升趋势（站上 60 日均线 + 量能健康） | 积极选股 |
| 震荡市 | 精选龙头 + 题材轮动 |
| 下降趋势 | **暂停选股**，仅观察 |

> 大盘下行时输出"暂不建议建仓"而非候选池。

### Layer 2：板块筛选（找强势赛道）

筛选标准（满足 ≥ 3 项进入候选板块）：

1. **相对强度（RPS）**：板块指数 / 大盘指数比值近 20 日上行
2. **资金流入**：行业 ETF 近 5 日净申购为正，主力净流入排名前 30%
3. **政策 / 景气催化**：明确产业政策、订单数据、价格信号
4. **量价配合**：板块指数放量上行或缩量回踩不破 20 日均线
5. **轮动位置**：处于"启动 / 主升"，避开"高位 / 退潮"

### Layer 3：个股多因子打分

对候选板块内的个股按以下维度打分（百分制）：

| 维度 | 权重 | 关键指标 |
|------|------|----------|
| 基本面质地 | 30% | ROE ≥ 12%、近 3 年净利复合增速 ≥ 15%、毛利率稳定 |
| 估值合理 | 15% | PE / PB 处近 5 年 30%–70% 分位，PEG ≤ 1.2 |
| 趋势强度 | 25% | 站上 60 日均线、20/60/120 多头排列、近 60 日涨幅强于板块 |
| 量价健康 | 15% | 上涨放量、回调缩量、无量价背离 |
| 资金面 | 10% | 北向 / 主力近 5 日净流入、机构持仓稳定或上升 |
| 风险扣分 | 5% | 商誉占净资产 > 30%、应收账款 > 营收 50%、质押 > 50%（每项扣 5–10 分） |

**入选阈值**：综合得分 ≥ 70 分；**同板块入选不超过 2 只**，避免过度集中。

> 行业差异化指标（银行 PB / 油运 TCE / 白酒批价 等）参见 stock-analysis §四，必要时替换"基本面质地"维度的具体指标。

## 四、风格化筛选模板

| 风格 | 关键过滤条件 |
|------|--------------|
| **龙头型** | 行业市占率前 3 + 近 1 年涨幅板块前 20% + 机构持仓上升 |
| **红利型** | 股息率 ≥ 4% + 近 5 年连续分红 + 自由现金流为正 + 资产负债率 ≤ 60% |
| **成长型** | 营收 YoY ≥ 30% + 净利润 YoY ≥ 25% + 研发费用率 ≥ 5% |
| **困境反转** | 行业景气度拐点 + 季度净利环比转正 + 技术面站上 120 日均线 |
| **高股息防御** | 股息率前 20% + Beta < 0.8 + 主营稳定 |
| **小市值黑马** | 市值 50–200 亿 + 营收近 8 季持续增长 + 机构调研次数上升 |

> 选择风格前**必须与用户确认**：投资周期（短/中/长）+ 风险偏好（保守/平衡/激进）。

## 五、主题 / 事件型选股

当用户给出明确主题（如"AI 算力""低空经济""红利""出海"）或事件（"降准""油价上涨""地缘冲突"）时，按以下结构筛选：

```
【主题/事件】<描述>
【作用链路】<事件 → 一级行业 → 二级标的 → 量化弹性>
【纯度评估】对每只候选标的：
  - 业务纯度：主题相关收入占比 ≥ X%
  - 弹性测算：事件下利润弹性 ±X%
【非主题剔除】
  - 蹭概念但无主营关联 → 剔除
  - 已在高位（近 1 年涨幅板块前 5%）且无新催化 → 剔除
【主题排序】纯度 + 估值 + 趋势强度 综合打分
```

> 主题型选股**风险更高**，必须主动提示"主题失效信号"（如政策反转、价格回落、龙头杀跌）。

## 六、候选池复盘（Pool Review）

当用户要求"前次推荐的标的现在怎么样了"时：

| 状态 | 判定 | 处置 |
|------|------|------|
| ✅ 已突破入场 | 站上推荐区间且放量 | 移交 `stock-buy` 确认买点 |
| ⏸ 仍在观察 | 价格在区间内震荡 | 维持候选池，更新有效期 |
| ⚠ 触发铁律 | 出现放量下跌、板块退潮 | 移出候选池，标注 @buy:Example1 触发 |
| ❌ 失效 | 跌破关键支撑 / 基本面恶化 / 进入黑名单 | 永久剔除并复盘错因 |

**复盘格式**：

```
【复盘时间】YYYY-MM-DD
【上次候选池版本】YYYY-MM-DD

| 代码 | 名称 | 推荐区间 | 当前价 | 涨跌幅 | 状态 | 处置 |
|------|------|----------|--------|--------|------|------|
| ...  | ...  | ...      | ...    | ...    | ✅/⏸/⚠/❌ | ... |

【方法论复盘】
- 命中率：X / N
- 主要失误：……
- 改进点：……
```

## 八、与其他 Skill 的衔接

- 上游：`stock-analyst` Agent 接收用户请求并分发本 Skill
- 下游：候选池产出后 →
  - 对 Top 1–3 调用 `stock-analysis` 做深度研判
  - 研判通过后调用 `stock-buy` 确认买点与仓位
  - 任何已持仓个股出现"放量下跌 / 板块退潮"信号 → 立即移交 `stock-sell`
- 复盘 → 写回 AGENT §八 作为方法论改进输入

## 九、禁止行为

- ❌ 在大盘下降趋势中产出"积极仓位"建议
- ❌ 输出 1 只"独苗推荐"（必须候选池形式）
- ❌ 把蹭概念但无主营关联的标的纳入主题池
- ❌ 把黑名单标的纳入候选池（无论得分多高）
- ❌ 越权给出具体买入价（属于 stock-buy 职责）
